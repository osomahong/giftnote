# gift-crawl-products

네이버 쇼핑 + 쿠팡파트너스 API로 상품을 수집하고, 자동 검증 후 반영.

## 현재 사용 플랫폼
- **coupang**: 쿠팡파트너스 API (검색 + 어필리에이트 링크 자동 생성)
- **naver**: 네이버 쇼핑 검색 (향후 공식 API 연동 예정)

## 향후 확장 예정
- kakao (gift.kakao.com) — Playwright 필요
- 29cm — Playwright 필요
- kurly — Playwright 필요
- ssg — SSR 가능, requests+BS4

## 쿠팡 상품 수집 워크플로우

### 1단계: 검색
`scripts/coupang-deeplink.ts` 실행.
쿠팡파트너스 API로 상품명 검색 → 후보 5개 반환.

### 2단계: 자동 검증 (3중 체크)
각 후보에 대해:
1. **브랜드 체크**: 원본 상품의 브랜드명이 후보 상품명에 포함되어 있는지 (필수)
2. **상품명 유사도**: 문자 단위 유사도 30% 이상 OR 키워드 겹침 50% 이상
3. **가격 범위**: 원본 가격의 ±50% 이내

### 3단계: 재시도
1차 검색어(상품명)로 실패 시, 2차 검색어(브랜드 + 키워드)로 재검색.
2차도 실패하면 해당 상품은 **매칭 실패로 표시하고 스킵**.

### 4단계: 적용
검증 통과한 상품만 MD 파일에 반영:
- affiliateUrl: 쿠팡 어필리에이트 링크
- image: 쿠팡 상품 이미지
- price: 실제 판매가
- sourceUrl: 쿠팡 상품 URL

### 5단계: 이미지 수집 + 교차 검증

#### 이미지 소스 우선순위
1. **다나와** (`img.danawa.com/prod_img/...`) — 상품명으로 다나와 검색 → pcode 이미지
2. **네이버 쇼핑** — 검색 결과에서 상품 이미지 추출
3. **공식 브랜드몰** — 브랜드 공식 사이트 이미지

#### 사용 금지 이미지
- **`ads-partners.coupang.com`** — 쿠팡 광고 배너 이미지 (상품 사진 아님, "특가는?" 등 프로모션 배너)
- 쿠팡파트너스 API의 `productImage`는 상품 이미지로 절대 사용하지 않음
- 쿠팡 API에서는 affiliateUrl, price, productName만 활용

#### 이미지 접근성 검증 (HTTP HEAD)
- 모든 이미지 URL에 대해 HTTP HEAD 요청으로 200 응답 확인
- 404/403 등 비정상 응답 → 해당 상품 이미지를 다른 소스에서 재검색
- 다나와 pcode가 가격비교 종료된 상품이면 이미지도 404가 됨 → 다른 pcode 사용

#### 교차 검증
- **같은 이미지 URL 중복 금지**: 컬렉션 내 2개 이상의 상품이 동일 이미지 URL을 공유하면 FAIL
- **이미지 URL에 제품 코드 포함 확인**: 다나와 이미지의 경우 pcode가 상품별로 고유해야 함
- 중복 발견 시: 해당 상품의 이미지를 상품명으로 재검색하여 교체
- 재검색도 실패하면: 이미지를 빈 문자열로 세팅 (프론트에서 플레이스홀더 표시)

### 6단계: 리포트
최종 검증 리포트 출력:
- ✅ 매칭 성공: 원본명 → 매칭된 쿠팡 상품명 + 가격
- ❌ 매칭 실패: 원본명 + 실패 사유
- ⚠️ 이미지 중복 수정: 어떤 상품의 이미지가 교체되었는지

## 검증 실패 시
- 해당 상품의 기존 링크/이미지를 유지 (덮어쓰지 않음)
- 리포트에 실패 사유 기록
- 사람에게 묻지 않고 자동으로 대안 탐색 → 플랫폼 전환까지 처리

## 필수 수집 필드 (누락 불가)
- **name**: 실제 상품명
- **brand**: 실제 브랜드명
- **price**: 실제 판매가 (number, KRW 원 단위)
- **image**: 상품 대표 이미지 URL (빈 문자열 절대 불가)
- **sourceUrl**: 실제 상품 페이지 URL
- **affiliateUrl**: 어필리에이트 딥링크

## evidence 출처 분류 규칙
- youtube: 유튜브 채널/영상
- community: 커뮤니티 (블라인드, 디시인사이드, 클리앙, 오늘의집, 네이버 카페 등)
- trend: 검색 트렌드, 판매 데이터 (네이버 트렌드, 구글 트렌드, 판매량)
- media: 언론/매거진 (GQ, 얼루어, 리빙센스 등)
- **블라인드는 SNS가 아님. type: "community"로 분류**

## 쿠팡 광고 문구
source=coupang인 상품에는 ProductCard에서 자동으로 표시:
"이 포스팅은 쿠팡 파트너스 활동의 일환으로, 이에 따른 일정액의 수수료를 제공받습니다"

## 출력
- products 배열 (Product 타입 준수, image 필드 필수)
- `content/products/{slug}.json` 원본 데이터 저장
- 검증 리포트 (터미널 출력)
