# gift-theme-planner

큐레이션 테마를 기획하고 slug, 태그, 메타데이터를 결정.
orchestrator의 1~3단계를 담당.

## 입력
- persona, interest, budget, occasion 조합
- 또는 "N개 기획해줘" → 자동으로 부족한 TPO 발굴

## 작업

### 1. 기존 콘텐츠 분석
content/collections/*.md를 읽고:
- 현재 TPO 커버리지 매트릭스 출력
- 큐레이터별 배정 현황
- 전체 상품 리스트 (중복 체크용)

### 2. 부족한 TPO 발굴
미충족 조합에서 주제 후보 선정:
- 기존 slug와 겹치지 않는지 확인
- 검색 볼륨이 높은 조합 우선

### 3. 큐레이터 배정

#### 성별 규칙 (절대)
- "여자친구 선물" → 남성 큐레이터만 (Eric, James, Leo, Owen)
- "남자친구 선물" → 여성 큐레이터만 (Clair, Hana, Mina, Yuna)
- "부모님/직장/친구/집들이" → 성별 무관
- 위반 시 자동으로 적합한 큐레이터로 교체

#### 균형 배분
- 가장 적게 배정된 큐레이터 우선
- 동일 큐레이터 3개 이상 연속 금지

#### 개성 매칭
| 큐레이터 | 강점 주제 |
|---|---|
| Eric | 테크, 가젯, 비교 분석 |
| Clair | 감성, 뷰티, 패키징 |
| James | 고가, 브랜드, 부모님/선배 |
| Hana | 가성비, 주방, 실용 |
| Leo | 스포츠, 아웃도어 |
| Mina | 인테리어, 디자인 |
| Owen | 감성, 의미, 관계 |
| Yuna | 트렌드, MZ, 직장 |

### 4. slug 생성
`{interest}-{target}-{budget}-{occasion}` 형식
예: `beauty-girlfriend-30k-anniversary`

### 5. 태그 선정 (3~5개)

#### 태그 압축 규칙
- **표준 태그 풀에서만 선택** (마구잡이 생성 금지)
- 동의어 중복 금지: "운동"과 "헬스" 중 하나만
- 범용 태그 금지: "실용적", "추천", "인기"

#### 표준 태그 풀
- **상황**: 생일, 집들이, 감사, 기념일, 명절, 졸업, 입학
- **대상**: 남자친구, 여자친구, 부모님, 직장동료, 친구
- **성별**: 남성, 여성 (공통이면 생략)
- **연령대**: 20대, 30대, 40대, 50대 (태그에는 세대만)
- **예산**: 1만원대, 2만원대, 3만원대, 5만원대, 10만원대
- **관심사**: 운동, 뷰티, 인테리어, 요리, 테크, 문화, 건강, 감성소품

### 6. ageGroup 세분화
frontmatter에는 "20대 초반", "20대 중반", "20대 후반" 등으로 기재.
태그에는 "20대"만 사용.

### 7. targets 작성 규칙
- description: 핵심 특성 1~2문장 + 인용/예시 1문장 (2~3문장 권장)
- 설명 문장과 인용구는 별도 문장으로 분리

## 출력
- frontmatter 초안 (YAML)
- 큐레이터 배정 사유
- 부족한 TPO 분석 결과
