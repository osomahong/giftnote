# gift-orchestrator

전체 큐레이션 파이프라인을 조율하는 오케스트레이터.
사람 개입 없이 기획부터 검증까지 자동 처리.

## 입력
- 콘텐츠 N개 생성 요청 (예: "콘텐츠 3개 작성해줘")
- 또는 특정 TPO 지정 (예: "30대 남성 집들이 선물 3만원대")

---

## 워크플로우 (6단계)

### 1단계: 기존 콘텐츠 분석

content/collections/*.md를 모두 읽고 현황 파악:

```
분석 항목:
- 현재 콘텐츠 수
- TPO 커버리지 매트릭스 (상황 × 대상 × 예산)
- 큐레이터별 작성 현황 (몇 개씩 배정되었는지)
- 전체 상품 리스트 (중복 체크용)
```

출력: 부족한 TPO 조합 목록 + 큐레이터 배분 현황

### 2단계: 부족한 TPO 발굴 + 주제 선정

표준 TPO 조합에서 아직 다루지 않은 것을 찾는다:

**상황(Occasion)**: 생일, 집들이, 감사, 기념일, 명절, 졸업
**대상(Person)**: 남자친구, 여자친구, 부모님, 직장동료, 친구
**예산(Budget)**: 1만원대, 2만원대, 3만원대, 5만원대, 10만원대
**관심사**: 운동, 뷰티, 인테리어, 요리, 테크, 문화, 건강, 감성소품

미충족 조합 예시:
- 30대 + 기념일 → 없음
- 40대 + 부모님 + 명절 → 없음
- 졸업 선물 전체 → 없음

주제 후보 3~5개를 리스트업하고, 기존 콘텐츠와 제목/슬러그가 겹치지 않는지 확인.

### 3단계: 큐레이터 배정

각 주제에 적합한 큐레이터를 배정한다:

**성별 규칙 (절대)**:
- 여자친구 선물 → 남성 큐레이터만 (Eric, James, Leo, Owen)
- 남자친구 선물 → 여성 큐레이터만 (Clair, Hana, Mina, Yuna)
- 부모님/직장/친구/집들이 → 성별 무관

**균형 배분**:
- 큐레이터별 작성 수를 확인하고, 가장 적게 배정된 큐레이터 우선
- 동일 큐레이터가 3개 이상 연속 배정되지 않도록

**개성 매칭**:
| 큐레이터 | 강점 주제 |
|---|---|
| Eric | 테크, 가젯, 비교 분석 필요한 주제 |
| Clair | 감성, 뷰티, 패키징 중요한 주제 |
| James | 고가 선물, 브랜드, 부모님/선배 |
| Hana | 가성비, 주방, 생활용품, 실용 |
| Leo | 스포츠, 아웃도어, 실사용 |
| Mina | 인테리어, 디자인, 공간 |
| Owen | 감성, 의미 있는 선물, 관계 |
| Yuna | 트렌드, MZ, 직장/동료 |

### 4단계: 상품 리서치

각 주제에 대해 상품을 **넉넉하게 7~10개** 후보로 리스트업한다.

**리서치 순서**:
1. 쿠팡파트너스 API로 관련 키워드 검색 (2~3개 키워드)
2. 결과에서 예산 범위 내 상품 선별
3. 이미지 확보 가능 여부 사전 확인 (다나와 pcode 존재?)

**중복 체크**:
- 기존 콘텐츠의 전체 상품 리스트와 비교
- 같은 상품이 50% 이상 겹치면 → 상품 교체
- 30% 이하 중복은 허용 (인기 상품은 여러 큐레이션에 등장 가능)

**최종 선정**: 7~10개 후보 중 5개 선정
- 이미지 확보 가능한 것만 (다나와 pcode 또는 공식몰 이미지)
- 브랜드 다양성 확보 (같은 브랜드 2개 이하)
- 가격 다양성 (예산 범위 내에서 상/중/하 분포)

### 5단계: 콘텐츠 생성

gift-theme-planner → MD 파일 작성 → 각 스킬 순서대로 실행:

```
1. gift-theme-planner → slug, 태그, frontmatter 초안
2. MD 파일 작성 (products, targets, editorial, faqs 포함)
3. coupang-deeplink.ts 실행 → 쿠팡 딥링크 자동 검증
4. verify-images.ts 실행 → 이미지 접근성 + pcode 브랜드 매칭
```

### 6단계: 최종 검증 + 빌드

```
1. 이미지 검증: verify-images.ts (HTTP 200 + pcode 매칭 + 중복 + 광고 차단)
2. 콘텐츠 품질:
   - 제목 60자 이내, 설명 160자 이내
   - 태그 3~5개 (표준 풀)
   - FAQ 4개
   - reason 50~80자, giftMessage 30~50자
   - evidence 2개 이상
   - 큐레이터 성별 적합성
3. 인덱스 빌드: build-recommendations.ts + build-tag-index.ts + build-sitemaps.ts
4. OG 이미지 생성: generate-og-images.ts
5. 빌드: next build
6. 커밋 + 푸시
```

## 핵심 규칙

### 크롤링 필수
상품 정보를 직접 작성하지 않음. 반드시 쿠팡 API 또는 네이버에서 실제 데이터 수집.

### 이미지 규칙
- `ads-partners.coupang.com` 절대 사용 금지
- 다나와 이미지 우선, 없으면 공식몰/네이버
- 이미지 미확보 상품은 큐레이션에서 제외
- 이미지 접근성(HTTP 200) + pcode 브랜드 매칭 필수

### 블라인드 = 커뮤니티
evidence에서 블라인드 출처는 type: "community"로 분류 (SNS 아님)

### 사람 개입 없이 자동 해결
- 쿠팡 검증 실패 → 대안 검색 → 네이버 전환 (자동)
- 이미지 불일치 → 재검색 (자동)
- 사용자에게 묻지 않고 최선의 대안으로 처리

### 큐레이터 성별 규칙
- 여자친구 선물 → 남성 큐레이터만
- 남자친구 선물 → 여성 큐레이터만
- 위반 시 자동 재배정

### 콘텐츠 가독성 규칙
- word-break: keep-all 전역 적용
- 인용구/대화체: 설명 문장과 분리
- reason: 50~80자. 기능적 장점 + 감성적 가치. 2문장 권장.
- giftMessage: 30~50자. 선물 메시지 한 줄.
- editorial body: 각 50~80자. 3문단.

### 태그 규칙
- 3~5개, 표준 풀에서만 선택
- 동의어 중복 금지
- 범용 태그 금지 ("실용적", "추천")
- ageGroup: "20대 초반/중반/후반" 세분화 (태그에는 "20대"만)

### 폰트 크기 기준 (PC / 모바일)
- 제목(h1): text-2xl / md:text-4xl
- 설명: text-base / md:text-lg
- 섹션 제목(h2): text-base / md:text-lg
- 본문: text-sm / md:text-base
- 상품명: text-base / md:text-lg

---

## 출력
- `content/collections/{slug}.md` 완성 파일
- `generated/recommendations.json` 업데이트
- `generated/tag-index.json` 업데이트
- `public/sitemap-*.xml` 사이트맵 업데이트
- `public/og/{slug}.png` OG 이미지
- 검증 리포트 (터미널 출력)
